## 1. v5.2 fairness — Level-bracket gate (15 min)

### 1.1 — Out-of-bracket BLOCKED (client-side pre-check, no popup)

- [ ] **Setup:** Mr_Boss Lv4 creates a 0.1 SUI wager. Sx is at Lv1, Lv2 OK; bump Sx to Lv6 if needed (`update_after_fight` via dev tool, or play through a fight).

- [ ] Click attempts (keyboard or dev-tools) surface SET_ERROR toast with the same bracket message — **no wallet popup**.

### 1.2 — Within-bracket happy path

- [ ] Bump Sx down (or Mr_Boss up) so both are within ±1 (e.g. Mr_Boss Lv4 + Sx Lv4 or Lv5).
- [ ] Accept button enables; click signs `request_accept_wager` (NOT `accept_wager`); the wager card flips to **PENDING_APPROVAL** with amber "Pending" badge on both screens.

### 1.3 — Chain-side bracket assertion (rare but pinned)

- [ ] In dev-tools console, manually craft a PTB that calls `request_accept_wager` with an out-of-bracket character (`buildRequestAcceptWagerTx(...)` with hardcoded mismatch). The tx aborts with `ELevelOutOfBracket (12)` — abort-humanizer toast: "You're outside the ±1 level bracket for this wager — try a wager from a player closer to your level."

> never started. Fixed via the `ownWagerIsTarget` gate guard +

> tests `testV52CreatorApprovesOwnTarget` (PROCEED) and
> `testV52CallerHasDifferentOwnWagerStillRollsBack` (preserved). Full

## 3. v5.2 fairness — Referee-liveness escape hatch (45+ min wall clock)

### 3.1 — Reclaim BLOCKED before 30 min (the abuse gate)

- [ ] Mr_Boss creates wager, Sx accepts, Mr_Boss approves — fight goes ACTIVE.
- [ ] In dev-tools, immediately call `buildReclaimStalledWagerTx(wagerId)` as Sx (the losing player would do this to escape).
- [ ] Tx aborts with **`EWagerNotStalled (19)`** — humanizer: "The wager isn't stalled yet — settlement window still open. Try again after 30 minutes from accept."
- [ ] **The fight arena banner does NOT render** (only appears after 30 min — gracefully hidden before).

### 3.2 — Reclaim WORKS at 30+ min for either participant

> **Live-testable as of 2026-05-30 (STEP 5 addendum).** Server now mirrors
> chain `WagerMatch.accepted_at` into `fight.wagerAcceptedAtMs` at fight-start
> (via `getWagerAcceptedAt` + `createFight` parameter). The banner activates
> off real fight state — no dev-tools workaround.

- [ ] **Setup the stall:** Mr_Boss creates wager, Sx accepts, Mr_Boss approves → ACTIVE.
- [ ] **Verify the wire** — open dev-tools, inspect `state.fight` (via Zustand devtool or `window.__sc_state` if exposed). Confirm `fight.wagerMatchId` is the on-chain wager id AND `fight.wagerAcceptedAtMs` is a real ms timestamp (close to `Date.now()` but anchored to the on-chain accept). This is the new wiring; if either is undefined, the banner can't activate.
- [ ] **Kill the server** before any settle tx fires (`pkill -f 'tsx src/index.ts'` or similar). Fight is now stranded ACTIVE on chain.
- [ ] Wait **31 minutes**. Optionally restart the frontend (don't restart server yet).
- [ ] On EITHER wallet's screen, navigate back into the fight (or it stays mounted) — the **"Referee unresponsive" banner** appears at the top automatically. (The component re-evaluates eligibility every 30s — no full reload needed for the boundary cross.)
- [ ] Click **Reclaim Stalled Wager** → signs `reclaim_stalled_wager`.
- [ ] On chain: both stakes refunded to their original depositors; status SETTLED; `WagerReclaimed` event with `reclaimed_by` set.
- [ ] Restart server. `[OrphanWager]` sweep on boot drops the now-SETTLED wager_in_flight row cleanly (`alreadySettled++`, not `cancelled++`).
- [ ] Verify Suiscan: both wallets +0.1 SUI net; TREASURY untouched (no platform fee on reclaim).

### 3.3 — Non-participant cannot reclaim

- [ ] During the 30+ min stall in 3.2 (or set up a fresh one), connect a third wallet (or use dev-tools).
- [ ] Call `buildReclaimStalledWagerTx(wagerId)` from the third wallet.
- [ ] Tx aborts with **`ENotWagerParticipant (20)`** — humanizer: "Only the two participants in this wager can reclaim its escrow."

### 3.4 — Permissionless backstop coexists (`cancel_expired_wager`)

- [ ] Same stall setup. Wait **10–25 min** (past `SETTLEMENT_TIMEOUT_MS = 10 min`, BEFORE `WAGER_RESOLUTION_TIMEOUT_MS = 30 min`).
- [ ] Any wallet can call `cancel_expired_wager` (the v5.1 permissionless 50/50 split) — works as before.
- [ ] After 30+ min, reclaim ALSO works if the permissionless backstop didn't fire. Both paths terminate safely; pin both can-coexist by running test 3.4 then 3.2 in different wagers.
