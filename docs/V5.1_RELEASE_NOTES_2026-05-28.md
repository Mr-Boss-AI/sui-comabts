## What's new in v5.1

### Contract layer (Move)

**Three new shared registries — chain-side gates for "one X per wallet":**

- **`CharacterRegistry`** — `Table<address, ID>`. `create_character` aborts on duplicate; `burn_character` clears the entry. Closes Block A Layer 3 (PTB-crafted duplicate-mint bypass).
- **`OpenWagerRegistry`** — `Table<address, ID>`. `create_wager` + `accept_wager` abort if caller has an open wager. All wager-completion paths (`cancel_wager` / `settle_wager` / `settle_tie` / `admin_cancel_wager` / `cancel_expired_wager`) remove the entry. Closes the server-down-mid-create orphan family AND the silent-accept-as-creator case.
- **`KioskRegistry`** — `Table<address, ID>`. `create_or_get_player_kiosk` replaces the v5.0 unconditional `create_player_kiosk` — returns existing kiosk_id if registered, otherwise creates one. Closes the 2026-05-20 phantom-empty-kiosk vector.

**New entry functions:**

- `arena::settle_tie(wager, registry, clock, ctx)` — TREASURY-only mutual-KO settlement. 100% refund to each side. No platform fee. Emits new `WagerTied` event. Closes the 2026-05-28 mutual-KO bug (incident wager `0xf2f3982266…`).
- `character::burn_character(admin, character, registry)` — admin-gated character delete; clears registry entry; lets the owner mint a fresh character on the same wallet.
- `character::update_after_fight_draw(admin, character, xp, clock)` — bumps the new `draws: u32` counter, applies XP, no rating change.
- `marketplace::create_or_get_player_kiosk(registry, ctx) → ID` — idempotent kiosk creation.

**Struct changes:**

- `Character` += `draws: u32` — chain-side W/L/D record.
- `Item` += `slot_type: u8` — 0=mainhand, 1=offhand, 2=both_hands. Enables chain-side enforcement of the GDD's shield/dual-wield/two-handed trinity. Frontend can drop the hardcoded `TWO_HANDED_NAMES` allowlist post-cutover.

**Equipment trinity enforcement** (`equipment.move`):
- `equip_weapon` aborts if item's `slot_type == both_hands` AND offhand DOF non-empty (`EOffhandOccupied`). Aborts if `slot_type == offhand` (`EItemNotMainhand`).
- `equip_offhand` aborts if item's `slot_type == both_hands` (`EItemNotOffhand`). Aborts if shield with `slot_type != offhand`. Aborts if current weapon's `slot_type == both_hands` (`EWeaponIsTwoHanded`).

**Item rarity stat budget enforcement** (`item.move::mint_item_admin`):
- Sum of all `*_bonus` fields + `max_damage` ≤ per-rarity budget.
- Budgets: Common=20, Uncommon=40, Rare=70, Epic=110, Legendary=160.
- Prevents admin minting a Legendary with stat-stack beyond its tier.

**Events added:**
- `WagerTied { match_id, player_a, player_b, refund_each }`
- `CharacterBurned { character_id, owner }`
- `KioskRegistered { kiosk_id, owner }`
- `DrawRecorded { character_id, owner, xp_gained, new_xp, new_draws }`

**Error codes added:**
- `EWalletAlreadyHasCharacter` (character.move 6)
- `ENotRegistered` (character.move 7)
- `EAlreadyHasOpenWager` (arena.move 11)
- `ERarityBudgetExceeded` (item.move 5)
- `EInvalidSlotType` (item.move 6)
- `EWeaponSlotTypeInvalid` (item.move 7)
- `EOffhandOccupied` (equipment.move 6)
- `EWeaponIsTwoHanded` (equipment.move 7)
- `EItemNotMainhand` (equipment.move 8)
- `EItemNotOffhand` (equipment.move 9)
